# Compose Todo Skill

Converts a feature spec into a **granular, file-level todo list** ordered by implementation
layer. Each todo is specific enough that an AI code editor can act on it without ambiguity.

---

## Tech Stack Reference

### Server (Backend)

```
Runtime:     Bun
Framework:   Elysia (HTTP + WebSocket)
ORM:         Drizzle ORM
DB:          PostgreSQL (Neon serverless driver) + PostGIS for geo
Queue:       BullMQ + Redis (ioredis)
Auth:        jose (JWT sign/verify)
Logger:      pino
Validation:  TypeBox (Elysia-native) + Zod (env/config only)
Templates:   Handlebars (notification templates)
IDs:         ULID (ulid package)
Scheduler:   cron package
```

**Path aliases:**
```
@core/*     → src/core/*
@modules/*  → src/modules/*
@infra/*    → src/infra/*
@db/*       → src/infra/db/*
```

**Module file structure:**
```
src/modules/[name]/
  entities/index.ts       ← EntitySchema definitions
  commands/index.ts       ← Command handlers
  queries/index.ts        ← Query handlers
  events/index.ts         ← Event emitters + listeners
  fsm/index.ts            ← State machine definitions
  jobs/index.ts           ← Scheduled & queue jobs
  index.ts                ← AppModule export
```

**Compose file structure:**
```
src/compose/[name]/
  index.ts                ← ComposeDefinition, hooks, rules, config
  entities/index.ts       ← Compose-level entity extensions (if any)
  fsm/index.ts            ← Compose-level FSMs (if any)
  routes/index.ts         ← Elysia route group
  jobs/index.ts           ← Compose-level scheduled jobs
```

**Infra file structure:**
```
src/infra/
  db/
    schema/[module].ts    ← Drizzle table definitions per module
    migrations/           ← auto-generated by drizzle-kit
    index.ts              ← db connection (Neon)
  queue/
    index.ts              ← BullMQ queue factory
    workers/[name].ts     ← Worker definitions per queue
  realtime/
    gateway.ts            ← Bun WebSocket RealTimeGateway impl
    bridge.ts             ← RealTimeBridge impl
  adapters/
    [type]/[provider].ts  ← e.g. notification/resend.ts, payment/stripe.ts
```

---

### Web (Frontend)

```
Base:          Vite + React + TypeScript
Routing:       TanStack Router (file-based, typed params)
Server State:  TanStack Query (caching, pagination, optimistic updates)
API Client:    Eden Treaty (@elysiajs/eden) — end-to-end types from Elysia
UI:            shadcn/ui + Radix UI + Tailwind CSS
Forms:         React Hook Form + Zod
Tables:        TanStack Table (headless, server-side pagination)
Charts:        Recharts
Real-Time:     Native WebSocket + Zustand
File Uploads:  Uppy (@uppy/core, @uppy/aws-s3)
Dates:         date-fns
```

**Web file structure:**
```
apps/admin/src/
  routes/
    [compose]/
      index.tsx           ← list page (table view)
      $id.tsx             ← detail/edit page
      new.tsx             ← create page
      $id.[action].tsx    ← action-specific pages/modals
  components/
    [compose]/
      [EntityName]Table.tsx
      [EntityName]Form.tsx
      [EntityName]Card.tsx
      [EntityName]StatusBadge.tsx
  hooks/
    use[EntityName].ts    ← TanStack Query hooks
    use[EntityName]Mutations.ts
  lib/
    api.ts                ← Eden Treaty client
    realtime.ts           ← WebSocket manager
    stores/[compose].ts   ← Zustand store slice
```

## Todo Generation Rules

1. **Order by dependency** — infra/DB first, then module, then compose, then API routes, then web
2. **One todo = one file action** — create, extend, or modify exactly one file per todo
3. **Be precise** — include the exact file path, export name, and what the file should contain
4. **Mark dependencies** — if a todo depends on a previous one, note `(requires #N)`
5. **Group by layer** with clear headings
6. **Include test todos** at the end of each layer for critical logic

---

## Standard Todo Structure

Output todos in this exact format:

```
## Layer N — [Layer Name]

- [ ] #N  [VERB] `[file/path.ts]`
          [One-sentence description of what this file does / what to add]
          Key exports: [ExportName, AnotherExport]
          (requires #M, #P)
```

Verbs: `CREATE`, `EXTEND`, `MODIFY`, `ADD TO`, `REGISTER IN`

## The 7 Layers (in order)

### Layer 1 — Infra: Database Schema

One todo per entity that needs a new or extended Drizzle table.

```
- [ ] CREATE `src/infra/db/schema/[compose].ts`
      Define Drizzle pgTable for [EntityName] with all fields from spec Section 3.
      Map geo fields to PostGIS geometry columns (geometry('Point',4326), etc.)
      Map money fields to integer columns (amount) + text column (currency).
      Key exports: [entityNameTable, EntityNameInsert, EntityNameSelect]

- [ ] ADD TO `src/infra/db/schema/index.ts`
      Re-export [entityNameTable] from [compose] schema file.

- [ ] CREATE `src/infra/db/migrations/[timestamp]_[compose]_[entity].sql`
      Run: bun drizzle-kit generate to produce migration.
      Then: bun drizzle-kit migrate to apply.
      NOTE: This is a CLI command, not a file to write — mark as "run migration" todo.
```

---

### Layer 2 — Module: Entity Schemas

One todo per EntitySchema declaration.

```
- [ ] CREATE/EXTEND `src/modules/[module]/entities/index.ts`
      Add EntitySchema for [EntityName] with all fields from spec Section 3.
      Register in EntitySchemaRegistry during module boot.
      Key exports: [EntityName]Schema
```

---

### Layer 3 — Module: FSMs

One todo per FSM defined in spec Section 4.

```
- [ ] CREATE/EXTEND `src/modules/[module]/fsm/index.ts`
      Implement [EntityName]FSM with states: [list].
      Wire guards using RuleEngine from SystemContext.
      Wire entry actions to emit DomainEvents via ctx.publish.
      Key exports: [EntityName]FSM
```

---

### Layer 4 — Module: Commands & Queries

Group command handlers and query handlers.

```
- [ ] EXTEND `src/modules/[module]/commands/index.ts`
      Register handler for [module.commandName].
      Payload: [field: type list from spec].
      Steps: [reference spec Section 5 steps].
      Emits: [event.name].

- [ ] EXTEND `src/modules/[module]/queries/index.ts`
      Register handler for [module.queryName].
      Returns: [PaginatedResult<EntityName> | EntityName].
      Filters: [list query params].
```

---

### Layer 5 — Module: Events & Jobs

```
- [ ] EXTEND `src/modules/[module]/events/index.ts`
      Subscribe to [event.name] from [source module].
      Handler: [what it does].

- [ ] EXTEND `src/modules/[module]/jobs/index.ts`
      Register scheduled job: [job.id], cron: [expression].
      Steps: [reference spec Section 10].
```

---

### Layer 6 — Compose: Hooks, Rules, Config

```
- [ ] CREATE `src/compose/[compose]/index.ts`
      Define ComposeDefinition with module list and moduleConfig.
      Register config: [key: value pairs from spec].

- [ ] EXTEND `src/compose/[compose]/index.ts`
      Add hook for [event.name] → [what it does per spec Section 7].
      (requires Layer 4 commands to be registered first)

- [ ] EXTEND `src/compose/[compose]/index.ts`
      Register business rules from spec Section 11.
      [rule-id]: [scope, guard, action summary].

- [ ] EXTEND `src/compose/[compose]/index.ts`
      Configure RealTimeBridge forwarding per spec Section 9.
```

---

### Layer 7 — API Routes (Elysia)

One todo per resource group.

```
- [ ] CREATE `src/compose/[compose]/routes/[resource].ts`
      Elysia route group for [Resource].
      Routes: [list from spec Section 8 — METHOD path permission].
      Auth: bearer token → identity.resolveSession → inject actor into ctx.
      Each handler: dispatch command or execute query via SystemContext.
      Key exports: [resource]Routes

- [ ] REGISTER `src/compose/[compose]/routes/index.ts`
      Mount [resource]Routes under /[compose]/[resource].

- [ ] REGISTER `src/app.ts` or `src/index.ts`
      Mount [compose]Routes on the Elysia app.
```

---

### Layer 8 — Notification Templates

```
- [ ] CREATE `src/modules/notification/templates/[compose]/[template-key].hbs`
      Handlebars template for [notification name].
      Variables: [list of template variables used].

- [ ] ADD TO `src/modules/notification/events/index.ts`
      Register NotificationTrigger for [event.name] → [template-key].
      RecipientExpr: [who receives it].
      Channel: [email|sms|push|in_app].
```

---

### Layer 9 — Web: API Hooks (TanStack Query)

```
- [ ] CREATE `apps/admin/src/hooks/use[EntityName].ts`
      useQuery hook: fetches [entity list or detail] via Eden Treaty.
      queryKey: ['[compose]', '[resource]', ...filters].
      Returns typed PaginatedResult<[EntityName]>.

- [ ] CREATE `apps/admin/src/hooks/use[EntityName]Mutations.ts`
      useMutation hooks for: [create, update, delete, action verbs from spec].
      Each mutation: calls Eden Treaty → invalidates relevant queryKeys on success.
      Optimistic update where UX demands immediate feedback.
```

---

### Layer 10 — Web: Components

```
- [ ] CREATE `apps/admin/src/components/[compose]/[EntityName]Table.tsx`
      TanStack Table with columns: [list key columns from spec entity].
      Server-side pagination via query params.
      Row actions: [edit, delete, action verbs].
      Column filters: [filterable fields from spec].

- [ ] CREATE `apps/admin/src/components/[compose]/[EntityName]Form.tsx`
      React Hook Form + Zod schema mirroring spec entity fields.
      Controlled inputs using shadcn/ui Form, Input, Select, DatePicker etc.
      Handles both create (no id) and edit (with id prefill) modes.
      Submission calls the relevant mutation hook.

- [ ] CREATE `apps/admin/src/components/[compose]/[EntityName]StatusBadge.tsx`
      Visual badge for FSM states defined in spec Section 4.
      Color mapping: [state → color per your design system].
```

---

### Layer 11 — Web: Routes (Pages)

```
- [ ] CREATE `apps/admin/src/routes/[compose]/index.tsx`
      List page for [EntityName].
      Loader: prefetch list query.
      Renders [EntityName]Table.
      Header: "New [EntityName]" button → navigates to /[compose]/new.

- [ ] CREATE `apps/admin/src/routes/[compose]/new.tsx`
      Create page.
      Renders [EntityName]Form in create mode.
      On success: navigate to /[compose]/$id.

- [ ] CREATE `apps/admin/src/routes/[compose]/$id.tsx`
      Detail/edit page.
      Loader: prefetch single entity by route param.
      Renders detail view + [EntityName]Form in edit mode.
      Action buttons: [FSM transitions from spec Section 4].

- [ ] CREATE `apps/admin/src/routes/[compose]/$id.[action].tsx`
      (one per non-CRUD action from spec Section 8 that needs its own page/modal)
```

---

### Layer 12 — Web: Real-Time Wiring

```
- [ ] EXTEND `apps/admin/src/lib/realtime.ts`
      Subscribe to channels: [list from spec Section 9] on mount.
      On event received: invalidate TanStack Query queryKeys or update Zustand store.

- [ ] CREATE/EXTEND `apps/admin/src/lib/stores/[compose].ts`
      Zustand slice for real-time UI state: [notification counts, live entity states, etc.]
```

---

## Optional Layers (include only if spec requires)

### Layer 13 — Analytics Dashboard Widgets (if spec includes analytics metrics)

```
- [ ] CREATE `apps/admin/src/components/[compose]/[MetricName]Chart.tsx`
      Recharts component for [metric from spec].

      Chart type: [Line|Bar|Area|Pie based on metric nature].
```

### Layer 14 — Integration Adapters (if spec requires new adapters)

```
- [ ] CREATE `src/infra/adapters/[type]/[provider].ts`

      Provider: [provider name].
      Methods: [list adapter methods from spec Section 12].

```

## Output Format Example

When generating todos for a real spec, produce them in this exact format:

```markdown
# Implementation Todos — [Feature Name] ([Compose])

> Spec version: 1.0.0 | Generated: [date]
> Total todos: N across 12 layers

---

## Layer 1 — Database Schema (N todos)

- [ ] #1  CREATE `src/infra/db/schema/crm.ts`
          Add `deals` pgTable with columns: id, organization_id, contact_id, title,
          pipeline_id, stage_id, owner_id, value_amount (integer), value_currency (text),
          expected_close_date (bigint), status (text), probability (integer),
          lost_reason (text), created_at (bigint), updated_at (bigint), deleted_at (bigint), version (integer).
          Key exports: dealsTable, DealInsert, DealSelect

- [ ] #2  ADD TO `src/infra/db/schema/index.ts`
          Re-export dealsTable from crm schema.

...

## Layer 2 — Entity Schemas (N todos)
...
```

Always number todos sequentially across all layers (not restarting per layer).
Always note `(requires #N)` when a later todo depends on an earlier one.
